# Contributing to Price Aggregator

Thank you for your interest in contributing to the Price Aggregator project! Here are some guidelines to help you get started.

## How to Contribute
1. **Fork the repository**: Click the 'Fork' button on the top right corner of the repo page.
2. **Clone your fork**: Clone your forked repository to your local machine:
   ```bash
   git clone https://github.com/yourusername/price-aggregator.git
   cd price-aggregator
   ```
3. **Create a branch**: Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b my-feature-branch
   ```
4. **Make your changes**: Implement your feature or fix, ensuring you follow the project's coding standards.
5. **Write tests**: Ensure that your code is well-tested. Add tests in the `__tests__` directory to cover your changes. Use Jest for unit and integration tests.
6. **Run tests**: Before committing your changes, run the tests to ensure everything is working:
   ```bash
   npm test
   ```
7. **Commit your changes**: Commit your changes with a clear message:
   ```bash
   git commit -m "Add my feature"
   ```
8. **Push your changes**: Push your changes to your fork:
   ```bash
   git push origin my-feature-branch
   ```
9. **Submit a pull request**: Go to the original repository and create a pull request.

## Code Standards
- Follow the TypeScript coding conventions used throughout the project.
- Use meaningful variable and function names to enhance code readability.
- Ensure proper indentation and spacing for clean code formatting.

## Testing Guidelines
- Write unit tests for each new feature or bug fix. Aim for a minimum of 80% test coverage.
- Use descriptive names for your test cases, indicating the expected behavior.
- Run tests using `npm test` before submitting your pull request to ensure all tests pass.

## Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. Please read it before contributing!

## Additional Contributions
If you have suggestions for improvements or additional features, feel free to open an issue or discuss it in the community forums!
