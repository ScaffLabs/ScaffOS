# Contributing to Event Bus Service

Thank you for considering contributing to the Event Bus service! We appreciate your help in making this project better.

## How to Contribute
1. **Fork the repository**: Click the fork button on the top right corner of the repository page.
2. **Clone your fork**: Use the command `git clone <your-fork-url>` to clone it locally.
3. **Create a feature branch**: Create a new branch for your feature using `git checkout -b feature/your-feature-name`.
4. **Make your changes**: Implement your feature or fix, ensuring you follow the coding standards outlined in the project. Please add comments explaining the purpose of your changes, especially for complex logic.
5. **Run tests**: Ensure that all tests pass before submitting your pull request. Use `npm test` to run the test suite and check coverage.
6. **Submit a pull request**: Push your changes to your fork and submit a pull request to the main repository. Include a description of your changes and any relevant issues it addresses.

## Code Style
- Follow the existing code style and structure. Consistency helps maintain readability.
- Write meaningful commit messages that describe the changes made.

## Testing
- Ensure all new features and bug fixes are covered by tests. Use Jest for unit testing, and ensure integration tests are also passing.
- Run `npm test` to verify that all tests pass before creating a pull request.

## Issues
- If you encounter any issues, please report them through the GitHub issues page. Provide as much detail as possible to help us understand the problem.

## Contact
For any questions or suggestions, please feel free to reach out to the maintainers via GitHub issues or discussions. Your feedback is valuable!
