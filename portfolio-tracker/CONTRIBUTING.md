# Contributing to Portfolio Tracker

Thank you for your interest in contributing to the Portfolio Tracker project!

## How to Contribute
1. **Fork the repository**: Create your own fork of the project on GitHub.
2. **Clone the forked repository**: Use `git clone` to copy your fork locally.
3. **Create a new branch**: Use `git checkout -b your-feature-branch` to create a new branch for your changes.
4. **Make your changes**: Implement your feature or fix a bug. Ensure you write tests for all new features.
5. **Run tests**: Before submitting, ensure all tests pass by running `npm test`.
6. **Commit your changes**: Use `git commit -m 'Describe your changes'` to commit your changes.
7. **Push to your fork**: Push your changes to your fork using `git push origin your-feature-branch`.
8. **Create a Pull Request**: Go to the original repository and create a Pull Request. Include a clear description of your changes and why they should be merged.

## Code of Conduct
This project adheres to a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Reporting Issues
If you encounter any issues, please report them by creating a new issue in the repository. Include as much detail as possible to help us resolve the issue.
